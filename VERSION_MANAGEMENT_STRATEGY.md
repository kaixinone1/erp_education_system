# 版本管理策略

## 概述
本策略定义了ERP系统的Git分支管理方案，确保代码版本的清晰管理和历史追溯。

## 分支结构

### 核心分支
1. **main** - 主分支
   - 包含稳定的生产代码
   - 所有发布版本都从这里创建
   - 只接受来自其他分支的合并请求

2. **backup** - 备份分支
   - 自动备份分支，每日自动更新
   - 用于快速恢复最近的备份
   - 由备份脚本自动管理

3. **archive** - 归档分支前缀
   - 用于存储历史版本快照
   - 命名格式：`archive/YYYY-MM-DD`
   - 定期创建，保留完整的版本历史

### 功能分支（可选）
- **feature/xxx** - 新功能开发
- **fix/xxx** -  bug修复
- **hotfix/xxx** - 紧急修复

## 分支管理流程

### 1. 主分支管理
- **提交策略**：保持主分支代码的稳定性
- **合并规则**：只接受经过测试的代码合并
- **保护机制**：在远程仓库中启用主分支保护

### 2. 备份分支管理
- **自动更新**：由备份脚本每日自动更新
- **内容同步**：与主分支保持同步
- **用途**：作为日常备份和快速恢复点

### 3. 归档分支管理
- **创建频率**：每月或每季度创建一次
- **命名规则**：`archive/YYYY-MM-DD`（如`archive/2024-01-30`）
- **内容**：包含创建时的完整代码状态
- **保留策略**：永久保留，作为历史版本记录

## 版本标签管理

### 版本号格式
- **格式**：`vX.Y.Z`
- **说明**：
  - `X`：主版本号（重大功能变更）
  - `Y`：次版本号（新功能添加）
  - `Z`：补丁版本号（bug修复）

### 标签使用场景
- **发布版本**：正式发布时创建
- **重大里程碑**：系统重大更新时创建
- **归档点**：与归档分支对应创建

## 归档分支创建方法

### 手动创建
```bash
# 切换到主分支
git checkout main

# 拉取最新代码
git pull origin main

# 创建归档分支
git checkout -b archive/$(date +"%Y-%m-%d")

# 推送归档分支到远程仓库
git push origin archive/$(date +"%Y-%m-%d")

# 创建版本标签（可选）
git tag -a v1.0.0 -m "版本 1.0.0 发布"
git push origin v1.0.0
```

### 自动创建
使用提供的 `create_archive_branch.ps1` 脚本自动创建归档分支。

## 版本管理最佳实践

### 1. 提交规范
- **提交信息格式**：
  ```
  <类型>: <描述>
  
  <详细说明>（可选）
  ```
- **提交类型**：
  - `feat`：新功能
  - `fix`：bug修复
  - `docs`：文档更新
  - `style`：代码风格调整
  - `refactor`：代码重构
  - `test`：测试相关
  - `chore`：构建或依赖更新

### 2. 分支操作规范
- **分支生命周期**：功能完成后及时合并并删除临时分支
- **合并请求**：创建合并请求时提供详细的变更说明
- **代码审查**：重要变更需要代码审查

### 3. 版本发布流程
1. **准备发布**：在主分支上进行最终测试
2. **创建标签**：为发布版本创建版本标签
3. **创建归档**：创建对应的归档分支
4. **发布通知**：记录发布内容和变更

### 4. 回滚策略
- **快速回滚**：使用备份分支快速恢复
- **版本回滚**：使用归档分支或标签回滚到特定版本
- **回滚流程**：
  ```bash
  # 查看版本历史
git log --oneline

  # 回滚到特定版本
git checkout main
git reset --hard <commit_hash>
git push origin main --force
  ```

## 归档分支管理

### 定期创建
- **推荐频率**：每月1日创建
- **创建时机**：在系统稳定运行后
- **命名示例**：
  - `archive/2024-01-01`
  - `archive/2024-02-01`

### 归档内容
- **完整代码**：包含所有配置文件和代码
- **版本信息**：对应版本的完整状态
- **历史记录**：保留完整的提交历史

### 归档使用场景
1. **历史版本查询**：查看特定时间点的代码状态
2. **问题追溯**：追溯历史问题的根源
3. **合规要求**：满足审计和合规要求
4. **灾难恢复**：在主分支损坏时恢复

## 工具和脚本

### 自动归档脚本
- **脚本**：`create_archive_branch.ps1`
- **功能**：自动创建归档分支并推送
- **使用方法**：
  ```powershell
  # 创建归档分支
  .\create_archive_branch.ps1

  # 指定归档日期
  .\create_archive_branch.ps1 -ArchiveDate "2024-01-30"
  ```
