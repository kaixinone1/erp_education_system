表名、字段名映射与管理方案。

核心原则：唯一性、描述性、一致性
表名规则：

主表：使用复数英文名词，清晰描述实体。例如：employees（员工主表）。
子表：采用“单数名词+_+复数名词”或直接描述关系。例如：employee_salaries（员工工资子表），salaries。
字典表：以_dict或_dictionary结尾。例如：person_tags（标签字典表）。
关系表：描述多对多关系，如employee_tag_relations（员工标签关系表）。
字段名规则：

全部使用英文小写字母、下划线分隔。
采用“名词_修饰词”结构，确保顾名思义。例如：employee_id（员工ID）、basic_salary_amount（基本工资额）。
绝对禁止在表名和字段名中使用中文、拼音、空格或特殊字符。
映射管理方案：配置驱动，统一翻译
这是解决混乱的关键。所有映射关系必须由核心配置文件field_mappings.json集中管理。

建立标准字段词典：
在field_mappings.json中，为每个业务实体（如员工、工资）预定义一套标准字段名。这是所有数据导入必须对齐的“靶心”。

{
  "standard_fields": {
    "employee": ["id", "name", "id_card", "gender_code"],
    "salary": ["id", "employee_id", "salary_type_code", "amount", "month"]
  }
}
配置映射规则：
为每一种可能上传的原始文件格式，配置其字段到标准字段的映射规则。

{
  "mappings": [
    {
      "source_file_pattern": "员工信息表*.xlsx",
      "rules": [
        {"source": "姓名", "target": "employee.name"},
        {"source": "身份证号", "target": "employee.id_card"},
        {"source": "性别", "target": "employee.gender_code", "converter": "gender_to_code"}
      ]
    }
  ]
}
智能预填充与人工审核：
在上传文件后的“数据预览与清洗工作台”中，系统应基于上述配置智能推断字段映射。用户必须在此界面人工审核并确认所有映射关系，这是保证数据质量不可绕过的一步。

数据关联方案：主外键严格约束
关联不准，统计必然出错。必须建立基于键的严格关联模型。

主表必须使用代理主键：
为employees这类主表设立一个与业务无关的、自增的数字型id字段作为主键。它是整个关联体系的基石。

子表必须使用外键关联：
所有子表（如salaries）必须包含一个指向主表id的字段，如employee_id。数据库应强制此外键约束，防止关联到不存在的主记录。

实施步骤
梳理与规范：盘点所有现有数据表，按照上述命名原则进行重命名和结构改造。
创建映射配置：分析所有数据来源（Excel、旧系统等），在field_mappings.json中为每一种来源建立完整的映射规则。
改造导入流程：所有数据必须通过“数据预览与清洗工作台”导入，强制经历“智能映射->人工审核”流程。
建立验证与监控：在table_schemas.json中为字段定义严格的数据类型和约束，在导入时进行验证。
针对复杂统计的标签解决方案
对于编制、数据来源等多变口径的统计，采用“标签体系”是治本之策。建立person_tags（标签字典）和employee_tag_relations（关联表）。统计时，通过组合标签（AND/OR）进行查询，即可获得精准结果，彻底摆脱对复杂字段结构的依赖。

注意：
采用“配置驱动”的智能字段映射方案，并严格执行主外键关联模型。

第一，所有表名和字段名的映射关系必须由核心配置文件field_mappings.json统一管理。你需要在其中为每个业务实体预定义一套标准字段名，并为每一种来源的原始文件格式配置其字段到标准字段的映射规则。

第二，所有数据必须通过“数据预览与清洗工作台”导入。系统会根据field_mappings.json的规则智能预填充映射关系，但你必须在此界面进行人工审核并最终确认，以此保证数据质量。

第三，主表（如employees）必须使用一个与业务无关的自增数字型id字段作为代理主键。所有子表（如salaries）必须包含一个指向该主键的外键字段（如employee_id），并在数据库层面建立强制关联。

第四，对于因编制、数据来源等多变口径导致的复杂统计问题，应建立标签体系。即创建person_tags字典表和employee_tag_relations关系表，通过组合标签进行查询，从而彻底摆脱对复杂字段结构的依赖。
