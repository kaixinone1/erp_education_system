# Git仓库权限控制指南

## 概述
本指南详细说明如何配置和管理ERP系统Git仓库的权限，确保代码安全和访问控制。

## 权限控制层次

### 1. 本地仓库权限
- **文件系统权限**：控制对本地.git目录的访问
- **Git配置**：本地Git用户信息和凭证管理
- **提交签名**：使用GPG密钥签名提交

### 2. 远程仓库权限
- **仓库访问权限**：控制谁可以访问远程仓库
- **分支保护**：保护重要分支不被随意修改
- **标签保护**：保护发布版本标签
- **CI/CD权限**：控制自动化流程的权限

## 本地仓库权限配置

### 文件系统权限设置

#### Windows系统
1. **找到.git目录**：在项目根目录中
2. **右键点击** → **属性** → **安全**标签页
3. **编辑**权限，设置以下权限：
   - **管理员**：完全控制
   - **开发人员**：修改权限
   - **其他用户**：读取权限
4. **应用**更改

#### 权限建议
- 限制对.git目录的写入权限，只允许授权用户修改
- 确保备份工具和脚本有足够的权限访问.git目录

### Git配置管理

#### 配置用户信息
```bash
# 设置全局用户信息
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 或设置仓库级用户信息
git config user.name "ERP System"
git config user.email "erp@example.com"
```

#### 凭证管理

##### Windows凭证管理器
1. **控制面板** → **用户账户** → **凭据管理器**
2. **Windows凭据** → **添加Windows凭据**
3. 输入：
   - **网络地址**：`git:https://github.com`
   - **用户名**：GitHub用户名
   - **密码**：Personal Access Token

##### Git凭证缓存
```bash
# 启用凭证缓存（默认15分钟）
git config --global credential.helper cache

# 设置缓存时间（小时）
git config --global credential.helper 'cache --timeout=3600'
```

### GPG签名配置

#### 生成GPG密钥
```bash
# 生成GPG密钥
gpg --full-generate-key

# 查看GPG密钥
gpg --list-secret-keys --keyid-format LONG
```

#### 配置Git使用GPG密钥
```bash
# 获取GPG密钥ID
gpg --list-secret-keys --keyid-format LONG

# 配置Git使用GPG密钥
git config --global user.signingkey YOUR_GPG_KEY_ID

# 启用自动签名
git config --global commit.gpgsign true
```

## 远程仓库权限配置

### GitHub权限设置

#### 1. 仓库访问控制
1. 登录GitHub，进入仓库页面
2. 点击**Settings** → **Access** → **Collaborators and teams**
3. 点击**Add people**或**Add teams**
4. 设置权限级别：
   - **Read**：只读访问
   - **Triage**：只读 + 管理问题和PR
   - **Write**：读写访问
   - **Maintain**：管理权限
   - **Admin**：完全管理权限

#### 2. 分支保护规则
1. 点击**Settings** → **Branches**
2. 点击**Add branch protection rule**
3. 配置规则：
   - **Branch name pattern**：`main`
   - **Require a pull request before merging**：启用
   - **Require approvals**：设置至少1个审批
   - **Require status checks to pass before merging**：启用
   - **Include administrators**：启用
   - **Restrict who can push to matching branches**：设置授权用户
4. 点击**Create**

#### 3. 标签保护
1. 点击**Settings** → **Tags**
2. 点击**Add tag protection rule**
3. 配置规则：
   - **Tag name pattern**：`v*`
   - **Restrict who can create matching tags**：设置授权用户
4. 点击**Create**

### GitLab权限设置

#### 1. 仓库访问控制
1. 登录GitLab，进入项目页面
2. 点击**Settings** → **Members**
3. 点击**Invite members**
4. 设置权限级别：
   - **Guest**：只读访问
   - **Reporter**：只读 + 管理问题
   - **Developer**：开发权限
   - **Maintainer**：维护权限
   - **Owner**：完全控制

#### 2. 分支保护
1. 点击**Settings** → **Repository** → **Protected branches**
2. 点击**Protect a branch**
3. 配置规则：
   - **Branch**：`main`
   - **Allowed to merge**：选择角色
   - **Allowed to push and merge**：选择角色
   - **Allow force push**：禁用
   - **Require code owner approval**：启用
4. 点击**Protect**

### Gitee权限设置

#### 1. 仓库访问控制
1. 登录Gitee，进入仓库页面
2. 点击**管理** → **成员管理**
3. 点击**添加成员**
4. 设置权限级别：
   - **观察者**：只读访问
   - **开发者**：开发权限
   - **维护者**：维护权限
   - **所有者**：完全控制

#### 2. 分支保护
1. 点击**管理** → **分支管理** → **保护分支**
2. 点击**添加规则**
3. 配置规则：
   - **分支名**：`main`
   - **可合并**：选择角色
   - **可推送**：选择角色
   - **需要审核**：启用
   - **禁止强制推送**：启用
4. 点击**确定**

## 分支保护策略

### 核心分支保护

#### main分支保护规则
- **强制代码审查**：所有合并必须经过代码审查
- **状态检查**：确保CI/CD测试通过
- **线性历史**：禁止非线性合并
- **管理员受限**：分支保护规则适用于所有用户，包括管理员
- **签名验证**：要求提交必须经过GPG签名

#### backup分支保护规则
- **自动更新**：只允许备份脚本更新
- **禁止手动修改**：防止手动修改备份分支
- **历史保留**：保留完整的备份历史

#### archive分支保护规则
- **只读**：归档分支一旦创建，禁止修改
- **标签关联**：每个归档分支关联对应的版本标签
- **历史完整**：保留完整的历史记录

### 分支保护实施

#### GitHub实施
1. 进入仓库 **Settings** → **Branches**
2. 为每个核心分支创建保护规则
3. 配置具体的保护选项
4. 测试分支保护规则

#### GitLab实施
1. 进入项目 **Settings** → **Repository** → **Protected branches**
2. 为每个核心分支创建保护规则
3. 配置权限和选项
4. 验证保护规则生效

## 安全最佳实践

### 1. 凭证管理
- **使用Personal Access Token**：替代密码认证
- **定期轮换凭证**：每3-6个月更新一次Token
- **最小权限原则**：Token只授予必要的权限
- **凭证加密**：使用安全的方式存储凭证

### 2. 访问控制
- **定期审查成员**：每月审查仓库成员和权限
- **及时移除离职人员**：确保离职人员无法访问
- **权限最小化**：根据实际需要分配最小权限
- **团队管理**：使用团队管理权限，而非个人权限

### 3. 代码安全
- **提交签名**：使用GPG密钥签名所有提交
- **代码扫描**：定期扫描代码中的敏感信息
- **依赖检查**：检查依赖包的安全漏洞
- **安全审查**：定期进行安全代码审查

### 4. 审计和监控
- **操作日志**：启用并审查仓库操作日志
- **异常检测**：监控异常的代码提交和分支操作
- **访问审计**：定期审计仓库访问记录
- **安全告警**：配置安全事件告警

## 权限 troubleshooting

### 常见权限问题

#### 1. 无法推送代码到远程仓库
**可能原因**：
- 没有推送权限
- 分支受到保护
- 凭证过期或无效
- 网络连接问题

**解决方法**：
- 检查仓库权限设置
- 检查分支保护规则
- 更新Git凭证
- 检查网络连接

#### 2. 无法合并Pull Request
**可能原因**：
- 缺少审批
- 状态检查失败
- 分支保护规则限制
- 冲突未解决

**解决方法**：
- 请团队成员审批
- 修复失败的状态检查
- 联系管理员调整分支保护规则
- 解决代码冲突

#### 3. 凭证管理问题
**可能原因**：
- 凭证缓存过期
- 凭证存储错误
- 多账户冲突

**解决方法**：
- 重新输入凭证
- 清理凭证缓存
- 配置 credential.helper

## 应急响应

### 安全事件处理

#### 1. 凭证泄露
**应急措施**：
- 立即撤销泄露的凭证
- 生成新的Personal Access Token
- 审查所有使用该凭证的操作
- 通知团队成员

#### 2. 未授权访问
**应急措施**：
- 立即移除未授权用户
- 审查未授权用户的操作
- 更改仓库访问凭证
- 增强访问控制

#### 3. 恶意代码提交
**应急措施**：
- 识别并移除恶意代码
- 撤销相关提交
- 检查其他分支是否受影响
- 加强代码审查流程
