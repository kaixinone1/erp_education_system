## 🎯 流程概览

系统采用**四步导入流程** + **三级验证机制**，确保数据导入的质量和安全性。

### 四步导入流程
```
文件选择归属 → 字段配置 → 数据验证 → 确认导入
     ↓              ↓           ↓          ↓
   步骤1          步骤2       步骤3      步骤4
```

### 三级验证机制
```
级别1 → 级别2 → 级别3
格式验证 → 业务逻辑 → 关联数据
```

```python
def get_english_field_name(self, chinese_name: str) -> str:
    # 1. 完整匹配常用词汇
    if chinese_name == '姓名':
        return 'name'
    if chinese_name == '身份证':
        return 'id_card'
    
    # 2. 部分匹配最长子串
    for key in self.common_words.keys():
        if key in chinese_name:
            return chinese_name.replace(key, self.common_words[key])
    
    # 3. 默认规则：拼音或直接转换
    return self._generate_english_name(chinese_name)
```

##### 🔵 级别1：格式与类型验证
**文件**：`backend/validators/level1_validator.py`

**验证内容**：
```python
def validate_level_1(table_name: str, data: List[Dict]) -> List[Dict]:
    errors = []
    
    for row_index, row in enumerate(data):
        for field in table_schema['fields']:
            # 1. 必填字段验证
            if not field['nullable'] and not row.get(field['field_name']):
                errors.append({
                    'row_index': row_index,
                    'field': field['field_name'],
                    'error_type': 'nullable_constraint',
                    'message': '必填字段不能为空'
                })
            
            # 2. 数据类型验证
            if field['data_type'] == 'INTEGER':
                try:
                    int(row[field['field_name']])
                except ValueError:
                    errors.append({...})
            
            # 3. 正则表达式验证
            if 'regex' in field['validation_rules']:
                pattern = field['validation_rules']['regex']['pattern']
                if not re.match(pattern, str(row[field['field_name']])):
                    errors.append({...})
            
            # 4. 长度验证
            if 'length' in field['validation_rules']:
                min_len = field['validation_rules']['length']['min']
                max_len = field['validation_rules']['length']['max']
                if not (min_len <= len(row[field['field_name']]) <= max_len):
                    errors.append({...})
            
            # 5. 范围验证
            if 'range' in field['validation_rules']:
                value = float(row[field['field_name']])
                if not (field['validation_rules']['range']['min'] <= value <= 
                       field['validation_rules']['range']['max']):
                    errors.append({...})
    
    return errors
```

##### 🟡 级别2：业务逻辑与唯一性验证
**文件**：`backend/validators/level2_validator.py`

**验证内容**：
```python
def validate_level_2(table_name: str, data: List[Dict], db: Session) -> List[Dict]:
    # 先执行级别1
    errors = validate_level_1(table_name, data)
    
    # 1. 唯一性验证（内存+数据库）
    for field in table_schema['fields']:
        if field['unique']:
            seen_values = set()
            
            for row_index, row in enumerate(data):
                value = row.get(field['field_name'])
                
                # 内存中检查
                if value in seen_values:
                    errors.append({
                        'error_type': 'unique_constraint',
                        'message': f'{field["display_name"]}必须唯一'
                    })
                seen_values.add(value)
                
                # 数据库中检查
                query = f"SELECT COUNT(*) FROM {table_name} WHERE {field_name} = :value"
                result = db.execute(query, {"value": value}).fetchone()
                if result[0] > 0:
                    errors.append({
                        'error_type': 'unique_constraint',
                        'message': f'{field["display_name"]}已存在于数据库中'
                    })
    
    # 2. 业务逻辑验证
    for row_index, row in enumerate(data):
        # 示例：出生日期不能晚于参加工作日期
        birth_date = row.get('birth_date')
        hire_date = row.get('hire_date')
        
        if birth_date and hire_date:
            if datetime.strptime(birth_date) > datetime.strptime(hire_date):
                errors.append({
                    'error_type': 'business_logic',
                    'message': '出生日期不能晚于参加工作日期'
                })
    
    return errors
```

##### 🟠 级别3：关联数据完整性验证
**文件**：`backend/validators/level3_validator.py`

**验证内容**：
```python
def validate_level_3(table_name: str, data: List[Dict], db: Session) -> List[Dict]:
    # 先执行级别2
    errors = validate_level_2(table_name, data, db)
    
    # 关联数据验证（外键检查）
    for field in table_schema['fields']:
        field_name = field['field_name']
        
        # 检查外键字段（如department_id）
        if field_name.endswith('_id') and field_name != 'id':
            related_table = field_name[:-3] + 's'  # 去掉_id加s
            
            for row_index, row in enumerate(data):
                foreign_key = row.get(field_name)
                
                if foreign_key:
                    # 查询关联表是否存在该ID
                    query = f"SELECT COUNT(*) FROM {related_table} WHERE id = :value"
                    result = db.execute(query, {"value": foreign_key}).fetchone()
                    
                    if not result or result[0] == 0:
                        errors.append({
                            'error_type': 'foreign_key_constraint',
                            'message': f'关联数据不存在于表{related_table}中'
                        })
    
    return errors
```

#### 4.3 后端导入处理
**文件**：`backend/routes/import_data.py`

**导入流程**：
```python
@router.post("/api/import-data")
def import_data(request: dict, db: Session = Depends(get_db)):
    # 1. 获取参数
    data = request.get("data", [])
    field_config = request.get("fieldConfig", [])
    table_name = request.get("tableName", "")
    display_name = request.get("displayName", "")
    is_final_import = request.get("isFinalImport", False)
    
    # 2. 数据转换
    transformed_data = []
    for row in data:
        transformed_row = {}
        for config in field_config:
            original_field = config.get("originalField")
            target_field = config.get("targetField")
            value = row.get(original_field)
            
            # 日期清洗
            if target_field in date_fields:
                value = clean_date(value)
            
            transformed_row[target_field] = str(value) if value else None
        transformed_data.append(transformed_row)
    
    # 3. 创建/重建表
    metadata = MetaData()
    columns = [Column('id', Integer, primary_key=True, autoincrement=True)]
    
    for field_name in transformed_data[0].keys():
        columns.append(
            Column(field_name.lower(), String(2000), nullable=True)
        )
    
    # 删除旧表（如果存在）
    if inspector.has_table(table_name):
        Table(table_name, metadata).drop(db.bind, checkfirst=True)
    
    # 创建新表
    table = Table(table_name, metadata, *columns)
    metadata.create_all(db.bind)
    
    # 4. 插入数据
    insert_stmt = table.insert()
    db.execute(insert_stmt, transformed_data)
    db.commit()
    
    # 5. 更新映射关系
    mapping_manager.save_mappings()
    
    return {
        "success": True,
        "message": f"导入成功，共{len(transformed_data)}条记录",
        "target_table": table_name
    }
```

## 🎯 三级验证机制详解

### 验证架构
```
┌─────────────────────────────────────────────────┐
│           ValidationDispatcher                   │
│              验证调度器                           │
└─────────────────────────────────────────────────┘
         ↓          ↓          ↓
    Level1      Level2      Level3
   格式验证    业务逻辑    关联数据
```

### 各级验证对比

| 特性 | 级别1 | 级别2 | 级别3 |
|------|-------|-------|-------|
| **验证内容** | 格式、类型、正则、长度、范围 | 唯一性、业务逻辑 | 外键关联 |
| **性能** | ⚡ 最快 | ⚡⚡ 中等 | ⚡⚡⚡ 较慢 |
| **数据库** | ❌ 不需要 | ✅ 需要 | ✅ 需要 |
| **适用场景** | 快速检查 | 深度检查 | 完整检查 |
| **典型错误** | 身份证格式错误 | 数据重复 | 部门不存在 |

### 验证级别选择策略

**级别1（快速验证）**：
- ✅ 适合数据量大的初步检查
- ✅ 重点关注数据格式和类型
- ✅ 无需数据库查询，性能最优

**级别2（深度验证）**：
- ✅ 适合正式导入前的全面检查
- ✅ 包含唯一性和业务逻辑
- ✅ 需要数据库查询

**级别3（完整验证）**：
- ✅ 适合重要数据的严格导入
- ✅ 包含所有关联数据检查
- ✅ 确保数据完整性

## 🐛 常见问题与解决方案

### 问题1：文件上传失败
**原因**：文件格式不支持或文件损坏  
**解决**：使用标准Excel格式，确保文件完整

### 问题2：字段映射不准确
**原因**：字段名不常见，自动映射失败  
**解决**：手动调整字段映射关系

### 问题3：验证级别选择错误
**原因**：不清楚各级别的作用  
**解决**：
- 快速检查用级别1
- 全面检查用级别2
- 严格检查用级别3

### 问题4：导入后找不到数据
**原因**：导航节点未正确添加  
**解决**：检查导航菜单或手动刷新页面

### 问题5：重复数据导入
**原因**：唯一性约束未生效  
**解决**：使用级别2或级别3验证
