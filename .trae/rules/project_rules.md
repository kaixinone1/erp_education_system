# 项目规则

## 绝对禁止硬编码

### 1. 禁止硬编码任何数据
- 包括默认值、测试数据、示例数据
- 包括字符串、数字、布尔值等任何类型的数据

### 2. 禁止硬编码任何字段值
- 如 `'退休原因': '正常退休'`
- 如 `'岗位': '教学岗位'`
- 如 `'职务': '教师'`

### 3. 禁止硬编码任何数字
- 如仪表盘统计数字 `23`、`5`、`18`
- 如分页大小、默认值等

### 4. 禁止硬编码任何字符串
- 如状态值 `'pending'`、`'completed'`
- 如类型值、分类值等

## 绝对禁止私自修改用户要求

### 1. 禁止私自修改用户的指示
- 禁止改变用户的原意
- 禁止添加用户没有要求的功能
- 禁止删除用户要求保留的内容

### 2. 禁止私自解释用户要求
- 必须按照用户的字面意思执行
- 有疑问时必须询问，不能自行判断

## 绝对禁止私自添加

### 1. 禁止私自添加字段
- 禁止在现有表中添加新字段
- 禁止使用 `ALTER TABLE ADD COLUMN`

### 2. 禁止私自添加表
- 禁止创建新表
- 禁止使用 `CREATE TABLE`

### 3. 禁止私自添加表名
- 禁止创建新的表名映射
- 禁止添加新的表名配置

### 4. 禁止私自添加数据
- 禁止写入测试数据
- 禁止写入示例数据
- 禁止创建测试脚本
- 禁止使用 `INSERT` 写入硬编码数据

## 正确做法

1. **从数据库查询真实数据**
   - 使用 `SELECT` 查询现有表
   - 使用现有字段，不添加新字段

2. **如果没有数据，保持为空**
   - 使用 `None` 或空字符串 `''`
   - 绝不使用默认值填充

3. **如果数据不存在，报错或提示用户**
   - 不假设数据存在
   - 不自动创建数据

4. **如果需要添加任何东西，必须询问用户**
   - 停止工作
   - 向用户说明需求
   - 等待用户明确许可

## 违反后果

如果发现自己即将违反任何规定，必须：
1. 立即停止
2. 询问用户如何处理
3. 等待用户明确指示

## 示例

### ❌ 错误（禁止）
```python
# 硬编码数据
'retirement_reason': '正常退休',
'岗位': '教学岗位',

# 私自添加字段
ALTER TABLE teachers ADD COLUMN 新字段 VARCHAR(100);

# 私自添加表
CREATE TABLE new_table (...);

# 私自添加数据
INSERT INTO table VALUES ('硬编码值');
```

### ✅ 正确
```python
# 从数据库查询，没有就为空
'retirement_reason': None,
'岗位': None,

# 询问用户后再操作
# "需要添加新字段吗？请确认。"
```

## 中文显示规则

### 1. 面向网页/用户界面
- 所有显示给用户的文本必须使用中文
- 包括按钮文字、标签、提示信息、标题等
- 包括表格列名、表单字段标签

### 2. 导出/下载功能
- 文件名必须使用中文
- 导出的数据字段名必须使用中文
- Excel/Word/PDF 中的表头必须使用中文
- 示例：`职工退休呈报表_2025年.xlsx`

### 3. 数据传递
- 从后端传递到前端的数据，字段名使用中文
- API 返回的 JSON 中，key 使用中文
- 示例：`{"姓名": "张三", "出生日期": "1965-06-07"}`

### 4. 日期格式
- 显示格式：`YYYY年M月D日`
- 示例：`2025年8月7日`

### 5. 语言设置
- Element Plus 组件必须使用中文语言包
- 日期选择器、分页等组件显示中文
