## 需求分析

1. **动态下拉菜单**：从任职状态字典读取选项，不是硬编码
2. **灵活配置**：每个状态对应不同的推送清单，可随时配置
3. **集中管理**：在 business_checklist 中配置触发条件和对应清单

## 设计方案

### 1. 修改 business_checklist 表结构
- 触发条件支持多个状态值
- 每个清单模板对应一个或多个触发状态

### 2. 前端动态加载任职状态
- 从 dict_dictionary 表读取任职状态选项
- 动态渲染下拉菜单

### 3. 后端智能匹配
- 根据状态值查询匹配的清单模板
- 支持一个状态触发多个清单

### 4. 配置示例
```json
{
  "清单名称": "退休业务清单",
  "触发条件": {"target_status": ["退休", "提前退休"]}
}
```
