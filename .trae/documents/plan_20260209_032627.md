# 职工退休呈报表导出功能重新设计方案

## 用户建议的设计思路
把Word文档作为背景，在背景上新建一个图层，然后逐个单元格比对，把数据填写到准确的位置。

## 新设计方案

### 1. 核心思路
- **保留原始模板文件不变**，不直接修改模板
- 创建一个新的Word文档，复制模板的结构和样式
- 通过**精确匹配单元格位置**来填入数据

### 2. 实现方式
```python
# 1. 打开原始模板（只读）
template_doc = Document(template_path)

# 2. 创建新文档
new_doc = Document()

# 3. 复制模板结构到新文档
for table in template_doc.tables:
    new_table = new_doc.add_table(rows=len(table.rows), cols=len(table.columns))
    new_table.style = table.style
    
    # 4. 逐个单元格比对并填入数据
    for row_idx, row in enumerate(table.rows):
        for cell_idx, cell in enumerate(row.cells):
            cell_text = cell.text.strip()
            new_cell = new_table.rows[row_idx].cells[cell_idx]
            
            # 根据单元格内容判断填入什么数据
            if cell_text == '姓名' or cell_text == '姓 名':
                new_cell.text = teacher_name
            elif cell_text == '性别' or cell_text == '性 别':
                new_cell.text = gender
            # ...其他字段
            else:
                # 保持原始内容
                new_cell.text = cell_text
```

### 3. 关键改进点

#### A. 精确匹配
- 使用**完全匹配**而不是包含匹配（`'姓名' in cell_text` → `cell_text == '姓名'`）
- 考虑空格差异（`'姓 名'` 和 `'姓名'`）

#### B. 数据映射表
定义清晰的数据映射关系：
```python
FIELD_MAPPING = {
    '姓名': 'teacher_name',
    '姓 名': 'teacher_name',
    '性别': 'gender',
    '性 别': 'gender',
    '出生年月': 'birth_date',
    # ...其他字段
}
```

#### C. 审批意见处理
- 审批意见中的姓名使用**占位符替换**
- 例如：`'经研究，同意 同志...'` → `'经研究，同意 {name} 同志...'`

### 4. 实施步骤
1. 删除现有的 `export-a3-report` API
2. 创建新的 `export-retirement-report` API
3. 实现模板复制和数据填充逻辑
4. 定义完整的数据映射表
5. 只导出Word文档（不处理PDF）

### 5. 优势
- 不破坏原始模板
- 精确控制数据填入位置
- 代码逻辑清晰，易于维护
- 避免合并单元格处理复杂问题

## 需要用户提供的信息
1. 完整的字段列表（所有需要填入数据的字段）
2. 模板中每个字段的准确文本（包括空格）
3. 审批意见中需要填入姓名的具体位置
