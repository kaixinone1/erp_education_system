## 问题分析

图一（推送清单模块）：使用 GenericDataView 通用数据管理器
- 显示原始表字段（ID、教师ID、清单ID等）
- 只有编辑/删除按钮
- 没有"处理"按钮打开清单详情

图二（待办工作）：使用定制的 dashboard 视图
- 显示格式化的任务名称
- 有"处理"按钮打开 ChecklistDrawer
- 更好的用户体验

## 解决方案

创建专用的 PushedChecklistView 组件，功能与待办工作一致：

1. **创建 PushedChecklistView.vue**
   - 从 API 加载 todo_work_items 数据
   - 格式化显示（新增XXX退休业务清单）
   - 添加"处理"按钮
   - 集成 ChecklistDrawer 抽屉

2. **修改路由配置**
   - 将 `/system/checklist/pushed` 指向新组件

3. **保持数据一致性**
   - 与待办工作使用相同的数据源
   - 相同的处理逻辑
