## 当前问题分析

模块管理中有三种类型：
1. **module** - 文件夹类型，用于组织子模块
2. **component** - 数据节点类型，挂载数据管理器
3. **report** - 报表节点类型

但当前实现存在以下问题：
- 模块类型判断不够清晰
- 文件夹类型模块不应该显示数据管理器
- 需要明确的区分规则

## 解决方案

### 1. 明确模块类型定义

**文件夹类型 (type: "module")**
- 特征：有 children 属性，用于组织子模块
- 行为：点击展开子菜单，不显示数据管理器
- 示例：系统管理、字典管理、教师管理

**数据节点类型 (type: "component")**
- 特征：有 component 属性，关联数据表
- 行为：点击显示数据管理器，可增删改查数据
- 示例：教师基础信息、教师学历记录

**报表类型 (type: "report")**
- 特征：有 report 属性，关联报表
- 行为：点击显示报表界面

### 2. 修改模块管理界面

在模块管理中增加类型说明和智能提示：
- 选择 "module" 类型时，隐藏数据表关联字段
- 选择 "component" 类型时，显示数据表关联字段
- 添加类型图标区分

### 3. 修改前端路由处理

在路由守卫中根据模块类型决定：
- module 类型：只渲染菜单，不加载数据组件
- component 类型：加载 GenericDataView 组件
- report 类型：加载报表组件

### 4. 添加类型校验

保存时校验：
- module 类型必须有 children 或为空
- component 类型必须有 component 和 table_name
- report 类型必须有 report 和 reportId
