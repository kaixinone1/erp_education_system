## 问题承认
我之前创建了多个混乱的路由和硬编码，导致系统无法正常工作。现在需要统一设计。

## 统一方案

### 1. 数据库设计
在 `document_templates` 表添加字段：
- `intermediate_table` - 关联的中间表名
- `intermediate_table_cn` - 中间表中文名

### 2. 统一 API
只保留一个数据获取 API：
- `GET /api/template-field-mapping/fill-data/{template_id}?teacher_id={id}`

逻辑：
1. 获取模板关联的中间表
2. 提取模板中的占位符
3. 如果有字段映射配置，使用映射查询
4. 如果没有映射，直接使用占位符名称作为字段名查询中间表

### 3. 前端统一
所有地方都调用同一个 API 获取数据

### 4. 配置方式
在模板管理页面，为每个模板配置：
1. 选择关联的中间表
2. （可选）配置字段映射

这样所有模板都能自动获取数据，无需为每个模板单独开发代码。
