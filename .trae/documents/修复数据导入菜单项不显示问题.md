## 问题分析

通过详细分析代码和配置文件，我发现菜单项不显示的根本原因是：

1. **前端无法获取导航数据**：
   - 前端控制台显示 `Failed to fetch` 错误，无法从 `http://127.0.0.1:8000/api/navigation-admin/tree` 获取导航数据
   - 后端服务器虽然正在运行，但可能存在网络连接问题或API实现问题

2. **导航配置与路由不一致**：
   - `navigation.json` 文件中数据导入模块的路径为 `/data-center/new-module`
   - 而路由配置中数据导入工作台的路径为 `/import/workbench`

3. **模块管理功能**：
   - 系统已实现模块管理功能，可以通过它来自动配置路由
   - 但需要确保后端API能够正确响应，并且前端能够正常加载导航数据

## 修复方案

### 1. 验证测试路由

- 确认 `http://localhost:5173/import/workbench` 路由是否能正常访问
- 验证数据导入功能的四步流程是否正常工作
- 测试后端数据导入API是否正常响应

### 2. 修复后端导航API

- 检查 `navigation_admin.py` 中的 GET API 实现
- 确保 `ConfigManager` 能够正确读取 `navigation.json` 文件
- 测试 `http://127.0.0.1:8000/api/navigation-admin/tree` API 是否能正常返回导航数据

### 3. 通过模块管理功能挂载数据导入模块

- 访问模块管理页面 (`/system/module-mgt`)
- 在数据中心模块下添加数据导入子模块
- 设置正确的路径为 `/import/workbench`
- 设置类型为 `component`，组件名称为 `DataImportWorkbench`
- 保存配置，确保后端API正确更新 `navigation.json` 文件

### 4. 验证导航菜单

- 刷新页面，检查左侧导航菜单中是否显示数据导入菜单项
- 点击菜单项，验证是否能正确跳转到 `/import/workbench`
- 测试其他菜单项是否正常显示和工作

### 5. 清理测试路由

- 验证数据导入功能通过模块管理配置的路由正常工作后，移除或保留测试路由作为备份

## 技术要点

1. **确保后端API可靠性**：
   - 修复导航数据获取API，确保前端能够正确加载导航菜单
   - 验证模块管理API能够正确更新导航配置

2. **保持配置一致性**：
   - 通过模块管理功能确保导航配置与实际路由保持一致
   - 验证模块管理功能能够自动配置正确的路由路径

3. **增强前端容错能力**：
   - 修改 `Sidebar` 组件，添加容错处理，确保即使API调用失败，系统也能正常运行

4. **分步骤验证**：
   - 先验证测试路由，确保功能正常
   - 再通过模块管理功能挂载到指定菜单下
   - 最后验证导航菜单和路由是否正常工作
