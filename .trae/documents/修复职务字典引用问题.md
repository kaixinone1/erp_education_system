## 设计方案

### 核心原则
1. **统一自增id** - 所有表使用系统自增id作为主键
2. **标准化关联** - 通过外键id关联，不直接存储code
3. **冗余显示字段** - 存储 `{字段}_名称` 方便查询
4. **支持复杂统计** - 便于多表JOIN和汇总

### 表结构规范

#### 字典表
```sql
- id (SERIAL PRIMARY KEY) - 系统自增id
- code (VARCHAR) - 业务代码
- name (VARCHAR) - 显示名称
- sort_order (INTEGER) - 排序
```

#### 子表
```sql
- id (SERIAL PRIMARY KEY)
- teacher_id (INTEGER) - 关联主表
- {字段}_id (INTEGER) - 关联字典id
- {字段}_名称 (VARCHAR) - 冗余存储名称
- {字段}_code (VARCHAR) - 冗余存储code
```

### 导入时自动处理
1. 创建字典表（id, code, name）
2. 导入字典数据（Excel"序号"→code，"职务"→name）
3. 创建子表（包含teacher_id和字典关联字段）
4. 通过身份证号码关联主表获取teacher_id
5. 通过code关联字典表获取{id, name}

### 统计查询支持
- 支持主表+多个子表JOIN
- 支持按字典项分组统计
- 支持复杂汇总报表
