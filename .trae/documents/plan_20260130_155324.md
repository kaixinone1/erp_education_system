## 添加模块管理菜单的实现方案

### 问题分析
用户要求在系统管理菜单中增加一个“模块管理”菜单，我需要修改导航数据结构来实现这个功能。

### 解决方案
修改 `frontend/src/components/layout/TagsView.vue` 文件中的导航数据结构，在系统管理菜单下添加模块管理子菜单。

### 实施步骤
1. **修改导航数据结构**：在 `navigationModules` 数组中的系统管理菜单下添加模块管理子菜单
2. **确保路由配置**：检查并确保路由配置中包含模块管理页面的路由
3. **测试功能**：确保新添加的菜单能够正常显示和访问

### 具体修改
在 `frontend/src/components/layout/TagsView.vue` 文件中，找到系统管理菜单的定义，然后在其 `children` 数组中添加模块管理子菜单：

```javascript
{
  id: '8',
  title: '系统管理',
  path: '/system',
  children: [
    {
      id: '8-1',
      title: '模块管理',
      path: '/system/modules'
    }
  ]
}
```

### 预期效果
- 系统管理菜单会显示为可展开/折叠的父菜单
- 展开后会显示模块管理子菜单
- 点击模块管理子菜单会导航到 `/system/modules` 路径
- 导航后会在标签栏中显示对应的标签
